Documentation Updates#

Ensure :orphan: tags are on the first line of rst files#

While creating the “new docs”, sections that are not yet fully fleshed out are tagged with an :orphan: tag. This tag MUST be on the first line or sphinx will still issue a warning that the RST file is not referenced in any toctree. Ensure when distribution statements are added/updated, the :orphan: tag stays on the first line.

modified: docs/source/new-docs/**/*.rst

Only include latest SCAT test dataset in check_system_requirements#
